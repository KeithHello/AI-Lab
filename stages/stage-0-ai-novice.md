## 🧠 知識點清單

### 1. 什麼是生成式 AI？

| 概念 | 一句話解釋 |
|---|---|
| **AI (Artificial Intelligence)** | 讓電腦模擬人類智慧的技術 |
| **機器學習 (Machine Learning)** | 從資料中學習規律，而非人工編寫規則 |
| **深度學習 (Deep Learning)** | 用多層神經網路處理複雜問題 |
| **生成式 AI (Generative AI)** | 能創造新內容（文字、圖片、程式碼）的 AI |
| **大語言模型 (LLM)** | 專注於理解和生成人類語言的超大規模模型 |

### 2. LLM 的核心概念（必懂 4 件事）

| 概念 | 核心問題 | 為什麼重要 |
|---|---|---|
| **Token** | 模型怎麼「讀」文字？ | 影響成本和輸入上限；中文 1 字 ≈ 1-2 Token，英文 1 字 ≈ 1 Token |
| **Context Window** | 一次能餵給模型多少內容？ | 最稀缺的資源；超出的內容會被「忘記」 |
| **Temperature** | 控制輸出的創意程度 | 0 = 保守精確、1 = 創意天馬行空 |
| **幻覺 (Hallucination)** | AI 為什麼會一本正經胡說八道？ | LLM 的本質是「預測下一個字」，而非「查詢事實」 |

### 3. 主流 AI 工具地圖

| 工具 | 一句話定位 | 何時使用 |
|---|---|---|
| **ChatGPT** | 全能通用助手 | 日常寫作、頭腦風暴、快速問答 |
| **Claude** | 長文分析專家 | 處理大量文件、技術文檔 |
| **Gemini** | Google 生態整合 | 搜尋相關、Google 全家桶 |
| **DeepSeek** | 高 CP 值開源模型 | 中文任務、預算有限 |
| **Perplexity** | AI 搜尋引擎 | 需要附來源的即時資訊 |
| **NotebookLM** | 個人文件問答 | 上傳 PDF 後提問 |
| **Copilot (GitHub/Microsoft)** | 寫程式助手 | 程式開發 |

### 4. AI 能做什麼 & 不能做什麼

**✅ 擅長的事**：
- 文字生成（文章、郵件、報告、文案）
- 摘要與翻譯
- 程式碼生成與除錯
- 頭腦風暴與創意發想
- 問答與解釋概念
- 簡單數據分析

**❌ 不擅長/要小心的事**：
- 保證 100% 事實準確（會產生幻覺）
- 即時事件（訓練資料有截止日期，除非聯網）
- 複雜數學計算（需要工具輔助）
- 替代人類判斷（最後決策應由人來做）

## 💪 練習項目

### 練習 1：初次對話（15 分鐘）

**目標**：與 AI 進行一次有意義的對話。

```
任務：請 ChatGPT 用一句話解釋「什麼是 AI Agent」
然後追問：「能用一個生活中的例子說明嗎？」
最後問：「它和一般聊天機器人有什麼不同？」
```

### 練習 2：模型對比實驗（20 分鐘）

**目標**：感受不同模型的能力差異。

```
用同一個問題測試至少 2 個模型：
「請幫我規劃一個週末台北一日遊，包含交通、美食、景點」

比較回覆的風格、細節程度、實用性，寫下你的觀察。
```

### 練習 3：Token 計算練習（15 分鐘）

**目標**：理解 Token 的概念。

```
在 OpenAI Tokenizer 貼上以下文字：
「AI 正在改變我們的工作方式，但重點不是 AI 能做什麼，
而是我們如何利用 AI 讓自己變得更好。」

記錄這段文字消耗的 Token 數，再試試英文版本看看差異。
```

### 練習 4：Prompt 對比練習（20 分鐘）

**目標**：直觀感受 Prompt 好壞的差異。

```
壞 Prompt：「寫一篇關於環保的文章」
好 Prompt：「你是一位環保專欄作家。請用 300 字以內，
為一般大眾寫一篇關於塑膠減量的短文，要有 3 個具體可行的建議。」

比較兩種 Prompt 的回覆品質，理解「給 AI 明確指令」的重要性。
```
