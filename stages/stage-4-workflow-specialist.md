# 🔗 Stage 4：流程自動化專家 (Workflow Specialist)

> **一句話定義**：企業數位轉型的核心推手。不迷信單一軟體，而是專精於打破「軟體孤島」，將企業碎片化的工具串聯成「免人工干預」的完全自動化流程。

### 你在這個階段的畫像

- 你發現許多企業的問題不是「缺軟體」，而是「軟體之間不溝通」
- 你可能在企業內部看到大量重複的人工作業（手動複製貼上、Excel 報表、Email 通知）
- 你想用自動化解放這些浪費的時間
- **這個階段的目標**：讓流程跑得比人快、比人準、而且 24 小時不休息

## 🧠 知識點清單

### 1. 自動化思維

#### 1.1 什麼值得自動化？

| 判斷標準 | 值得自動化 | 不值得自動化 |
|---|---|---|
| **頻率** | 每天或每週發生 | 一年一次 |
| **規則性** | 規則明確，決策邏輯清晰 | 每次都要人為判斷 |
| **規模** | 大量重複操作 | 小量一次性任務 |
| **錯誤成本** | 人為出錯代價高 | 出錯影響不大 |
| **ROI** | 自動化開發時間 < 半年人工時間 | 開發比人工還久 |

#### 1.2 自動化設計流程

```
1. 觀察 ──→ 2. 記錄 ──→ 3. 拆解 ──→ 4. 建構 ──→ 5. 測試 ──→ 6. 監控
    │            │           │           │           │           │
發現重複    寫下每一步    把流程拆成    用工具實現   跑真實數據   確保穩定
的人工流程    的動作      獨立步驟      每個步驟      驗證結果    持續運轉
```

### 2. 核心自動化工具

| 工具 | 類型 | 一句話 | 適合場景 |
|---|---|---|---|

### 3. 自動化核心技術點

#### 3.1 Webhook 非同步觸發

```
┌──────────┐    事件發生     ┌──────────────┐
│  服務 A  │ ──────────────→ │  自動化平台  │
│ (Gmail)  │  收到新郵件時    │  (Make/n8n)  │
│          │  發送 Webhook   │  觸發工作流  │
└──────────┘                 └──────┬───────┘
                                    │
                               ┌────▼───────┐
                               │   服務 B    │
                               │ (Google Sheet)
                               │  自動記錄  │
                               └────────────┘
```

#### 3.2 JSON 資料解析與條件分支

```json
{
  "order": {
    "id": "ORD-12345",
    "customer": "張小明",
    "items": [
      {"name": "商品A", "qty": 2, "price": 500},
      {"name": "商品B", "qty": 1, "price": 1200}
    ],
    "total": 2200,
    "status": "paid"
  }
}
```

**在自動化中處理 JSON**：
- 解析（Parse）：取出需要的欄位（如 `order.total`）
- 條件分支（Router）：`status == "paid"` → 觸發出貨流程；`status == "pending"` → 發送催款通知
- 轉換（Transform）：將 JSON 轉成其他格式（如：CSV、Email 內容）

#### 3.3 常見企業系統串接

| 系統類型 | 代表工具 | 常見串接場景 |
|---|---|---|
| **通訊** | LINE、Slack、Teams | 自動通知、Bot 回覆、訊息轉發 |
| **郵件** | Gmail、Outlook | 自動分類、自動回覆、附件處理 |
| **文書** | Google Sheets/Drive、Notion | 自動填表、報表生成 |
| **CRM** | Salesforce、HubSpot | 商機自動追蹤、客戶標籤 |
| **電商** | Shopify、蝦皮 | 訂單管理、庫存同步 |
| **ERP** | SAP、Oracle | 採購單自動生成、發票處理 |

#### 3.4 錯誤處理與監控

| 元件 | 說明 |
|---|---|
| **Error Handler** | 某步驟失敗時的備案（如：寄 Email 失敗 → 改發簡訊） |
| **Retry 機制** | 自動重試失敗步驟（如：API 逾時重試 3 次） |
| **通知機制** | 流程異常時通知負責人 |
| **Log 記錄** | 每一步的輸入輸出都要記錄，方便排查 |

## 💪 練習項目

### 練習 1：第一個自動化流程（2 小時）

**目標**：用 Make.com 串接 Gmail → Google Sheets。

```
流程設計：
觸發器：Gmail 收到新郵件（特定標籤）
  ↓
篩選器：只處理主旨包含「訂單」的郵件
  ↓
動作：提取寄件者、主旨、日期，寫入 Google Sheets 新的一列
  ↓
動作：發送 Slack 通知「新訂單已記錄」

驗證：寄一封測試郵件，確認流程跑通。
```

### 練習 2：跨平台自動化（3 小時）

**目標**：串接 LINE + Google Sheets + AI。

```
流程設計：
觸發器：收到 LINE 訊息
  ↓
條件判斷：訊息內容包含「查詢庫存」
  ↓
動作：查詢 Google Sheets 中的庫存表
  ↓
動作：呼叫 ChatGPT API 生成回覆
  ↓
動作：透過 LINE Bot 回覆使用者
```

### 練習 3（專案 A）：自動化商機追蹤與合約生成（5 小時）

**目標**：為一個虛擬企業打造端到端自動化流程。

```
業務場景：
- 業務人員在 CRM（如 HubSpot）建立新商機
- 系統自動從 Google Sheets 抓取產品報價
- 自動生成合約（用 Google Docs 模板 + AI 填充內容）
- 自動發送合約 Email 給客戶
- 客戶簽回後，自動更新 CRM 狀態 + 通知業務

技術要點：Webhook 觸發、JSON 資料轉換、AI 文字生成、多平台串接
```

### 練習 4（專案 B）：多平台客服派單與庫存同步（5 小時）

**目標**：設計一個跨平台的自動化客服系統。

```
業務場景：
- 客戶在 LINE / Email / 官網表單 任一管道提出問題
- 系統自動分類問題類型（退貨 / 詢問 / 客訴）
- 根據分類指派給對應的客服人員（Slack 通知）
- 如是退貨相關，自動查詢庫存系統確認數量
- 全部對話記錄自動歸檔到 Google Sheets
```

## 🏅 通過標準

**硬性指標**：

- [ ] 獨立交付 **2 個 Hands-on 實戰專案**
- [ ] 每個專案必須是針對真實企業痛點的「端到端完全自動化流程」
- [ ] 流程必須包含：觸發器 + 條件分支 + 多重資料轉換 + 錯誤處理 + 通知機制
- [ ] 流程必須涵蓋至少 3 個不同平台的串接（如：Email + Sheets + LINE）

**軟性指標**：

- [ ] 能獨立診斷一個自動化流程為什麼「卡住」並修復
- [ ] 能向非技術人員解釋自動化流程的運作邏輯
- [ ] 能估算一個自動化專案的開發時間和節省的人力成本
