## 每個 Stage 手冊包含 8 個區塊

| # | 區塊 | 內容 |
|---|---|---|
| 1 | 🎯 等級定位 | 名稱、定義、對應 Roadmap Phase、建議時間 |
| 2 | 📐 前置條件 | 進入此 Stage 前必須完成什麼 |
| 3 | 🧠 知識點清單 | 完整知識體系（核心概念 + 關鍵技術 + 工具鏈） |
| 4 | 🛠️ 技能要求 | 硬技能、軟技能、推薦工具 |
| 5 | 💪 練習項目 | 3-6 個從簡到難的實戰練習（含時間和目標） |
| 6 | 🏅 通過標準 | 硬性指標（不可妥協）+ 軟性指標 |
| 7 | 🔝 升階條件 | 關鍵問題 + 下一步路線選擇 |
| 8 | 📚 學習資源 | 推薦教材、開源專案、論文、社群 |

## 7 階段總覽

| Stage | 名稱 | 關鍵特徵 |
|---|---|---|
| 0 | AI 新手村 | 消除恐懼，4 個練習，1-2 天 |
| 1 | AI 探索者 | Prompt 工程，Function Calling 初體驗，1 週 |
| 2 | AI 實踐者 | Vibe Coding + Coze/Dify，4 個練習，2 週 |
| 3 | 初階建造者 | LangChain Agent + 全棧開發，6 個練習含 2 個專案 |
| 4 | 自動化專家 | Make/n8n + 跨平台串接，4 個練習含 2 個專案 |
| 5 | 高階架構師 | RAG + Multi-Agent + LangGraph + MCP，6 個練習含 3 個專案 |
| 6 | 自我進化架構師 🆕 | Reflexion + DSPy + GRPO + Meta-Agent，4 個練習 + 3 方案考核 |

## 設計原則

1. **做中學** — 每個階段以可運行的 Project 為終點
2. **雙體系互補** — 等級制度定義「誰」，路線圖定義「學什麼」
3. **梯隊互助** — 高等級帶低等級，構建學習生態
4. **硬性通過標準** — 每個等級有不可妥協的硬指標
5. **多路線支援** — 非技術/技術/雙刀流三條升級路徑
